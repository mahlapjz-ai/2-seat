👥 用户管理说明

• 新注册用户默认角色为"读者"，需在后台审核并手动提升为辅专或管理者。

• 辅专需分配管辖楼层（可多选），仅能操作所管楼层的拍照、上传、删除等功能。

• 辅专可以查看所有楼层的图片缩略图和拼接预览，但不能操作无权限楼层。

• 协助者（义工）的权限有有效期，过期后自动降级为读者。

📌 协作密码规则

生成规则

• 每天 08:00 自动生成第一批普通密码（5个楼层各1个），过期时间为当天 12:30。

• 每天 13:30 自动生成第二批普通密码（5个楼层各1个），过期时间为当天 17:30。

• 默认最大使用次数为 10 次，可在生成时修改。

• 手动生成的普通密码与自动生成的密码互不影响。

• 手动生成时可选择创建"临时"密码：有效期完全由管理员指定（留空则默认当天 23:59:59），不受时段限制，适用于测试或特殊情况。

使用与限制

• 吊销密码后，已使用该密码登录的协作者权限立即降级为读者权限。

辅专获取方式

• 辅专可在 PWA 的"设置"→"协作密码"中，查看自己管辖楼层的当天协作密码二维码。

• 临时密码同样出现在 PWA 的"协作密码"页面中，位于五楼下方。

扫码权限规则

• 普通密码在 08:00～12:30 扫码 → 权限至当天 12:30 到期

• 普通密码在 13:30～17:30 扫码 → 权限至当天 17:30 到期

• 普通密码在其他时间扫码 → 拒绝授权（不授予权限）

• 临时密码在有效期内任意时间扫码 → 按管理员设定的过期时间授予权限

• 临时密码超过有效期扫码 → 密码失效，无法使用

⚙️ 全局设置说明

• "读者视图开关"：开启后，角色为"读者"的用户登录后可查看座位占用状态（无照片）；关闭后读者看不到任何内容。

• "功能开关"区域：控制 PWA 中特定功能的显示与隐藏。默认全部关闭，仅所有者角色不受限制。

• 修改设置后即时生效，无需刷新 PWA。

可选：设置每日自动清理（需在 Supabase SQL 编辑器中执行）

SELECT cron.schedule('downgrade-expired-assistants', '0 0 * * *', $$ UPDATE public.users SET role = 'reader', managed_floors = NULL, assistant_expires_at = NULL WHERE role = 'assistant' AND assistant_expires_at < now(); $$);

📊 报表规则

• 每天 21:30 自动生成当日报表，也可点击"手动生成今日报表"立即生成。

• 报表数据基于当天图片统计：1 张图 → 填"×"，2 或 3 张图 → 填"△"。

• 早班责任人统计 08:30-13:14 的操作者，晚班责任人统计 13:15-21:20 的操作者。

• 报表存储在云端，可在线预览或下载，支持删除和按日期筛选。
